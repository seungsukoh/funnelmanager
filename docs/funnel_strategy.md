# 퍼널 전문가 검토 메모

## 결론

현재 MVP는 `응답 1건 -> 조건 판단 -> 템플릿 선택/발송` 구조다. 다음 제품 단계는 단발 발송기가 아니라, 리드 상태를 저장하고 단계별로 다음 발송을 예약하는 상태 기반 drip campaign 엔진이다.

## 1차 MVP에 필요한 퍼널 기능

### 단계형 drip campaign

- 캠페인은 여러 단계로 구성되어야 한다.
- 예: `Step 1 첫 안내 -> 2일 후 Step 2 리마인드 -> 5일 후 Step 3 마지막 안내`
- 각 단계는 템플릿, 대기 시간, 발송 조건, 스킵 조건을 가진다.
- 각 리드가 현재 몇 단계에 있는지 저장해야 한다.

### 리드 상태

최소 상태:

- 신규
- 육성중
- 관심있음
- 전환됨
- 제외
- 수신거부

`전환됨`, `제외`, `수신거부`는 전역 종료 조건으로 처리한다.

### 태그

상태와 태그는 분리한다.

- 상태: 현재 퍼널 위치
- 태그: 리드의 속성

예:

- B2B
- 가격문의
- 세미나참석
- 고관심
- 비활성
- VIP

룰 결과로 태그를 자동 부여하고, 이후 조건에서 재사용할 수 있어야 한다.

### 조건 우선순위

- 1차 MVP는 위에서 아래로 평가하고 첫 매칭 룰을 사용한다.
- 운영자가 이해하기 쉽고 dry-run에서 설명하기 좋다.
- 부수 액션은 후속 단계에서 여러 룰 동시 실행으로 확장한다.

### 스킵/종료 조건

전역 종료 조건:

- 수신거부
- 이메일 반송
- 전환 완료
- 운영자 제외
- 필수 동의 없음

단계별 스킵 조건:

- 이미 해당 템플릿 발송됨
- 특정 태그 보유
- 특정 상태가 아님
- 필수 필드 누락
- 최근 N일 내 발송됨

### 운영자용 룰 표현

JSON은 내부 저장 포맷으로 두고, UI에서는 문장형으로 보여준다.

예:

```text
상태가 신규이고
관심분야가 "가격"을 포함하면
태그 "가격문의"를 추가하고
2일 후 가격 안내 메일 발송
```

각 룰에는 이름, 설명, 우선순위, 활성/비활성, 테스트 결과가 필요하다.

## 후순위 기능

- 복잡한 visual journey builder
- AI 세그먼트 추천
- 고급 리드 스코어링
- A/B 테스트
- SMS/카카오/전화 태스크
- 고급 attribution 분석
- CRM 전체 기능
- 실시간 행동 추적 트리거
- 고급 권한 관리

## 추천 구현 순서

1. 리드 모델에 `status`, `tags`, `campaign_step`, `next_send_at`, `suppression` 개념 추가
2. campaign/step 구조 도입
3. 룰 우선순위와 match trace 추가
4. 스킵/종료 조건 추가
5. 발송 큐와 리드 타임라인 추가
6. JSON 룰 위에 운영자용 문장형 룰 표현 추가

## 현재 반영 상태

- `status`, `tags`, `campaign_step`, `next_send_at` 저장 구조 추가
- `전환됨`, `제외`, `수신거부` 상태는 전역 종료 조건으로 처리
- `next_send_at`이 미래인 리드는 아직 발송 시점이 아니므로 스킵
- 룰 action에서 `set_status`, `add_tags`, `set_step`, `next_send_after_days`를 선언할 수 있음
- 상태 변경은 dry-run이나 테스트 발송에서는 적용하지 않고, 실제 발송 성공 시에만 적용
- `steps` 설정을 통해 단계형 drip campaign을 선언할 수 있음
- 처리 결과는 `lead_timeline.jsonl`에 리드별 타임라인 이벤트로 기록
