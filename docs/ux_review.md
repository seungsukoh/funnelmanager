# UX/UI 검토 메모

## 검토 결과

기존 화면은 기능은 충분하지만, 처음 사용하는 사람이 보기에는 버튼이 많고 실행 순서가 분산되어 있었다. 특히 Gmail/Google 관련 버튼이 상단에 한꺼번에 노출되어 “무엇을 먼저 눌러야 하는지”가 명확하지 않았다.

## 개선한 점

- 첫 화면에 `오늘 진행 순서`를 추가했다.
- 작업을 `명단 확인 -> 단계별 메일 -> 발송 승인 -> 미리보기 -> Gmail 결과` 순서로 배치했다.
- 각 단계 카드에 현재 상태와 바로 실행할 버튼을 함께 배치했다.
- 다음에 해야 할 작업을 `다음 작업` 문구와 강조 테두리로 표시한다.
- 기존 탭 구조는 유지하되, 세부 확인은 탭에서 하고 주요 실행은 첫 화면 순서 카드에서 하도록 정리했다.
- 비공개 Google Sheet 설정은 `Gmail 확인` 탭의 안내 박스에서 준비 상태를 확인하도록 했다.
- CLI에 있던 Gmail 발송 준비 파일 생성 단계를 웹 버튼으로 옮겨 운영 흐름이 끊기지 않도록 했다.

## 남은 UX 개선 후보

- 실제 Google 계정 연결 후 오류 문구를 사용자 표현으로 더 다듬기
- Google Sheet 업로드까지 앱에서 자동화해 CSV 파일 이동 단계를 줄이기
- 명단 파일/메일 흐름 파일을 직접 선택하는 파일 선택 UI 추가
- 확인 필요 항목만 모아 다시 처리하는 화면 추가
