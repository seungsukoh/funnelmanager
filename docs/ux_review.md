# UX/UI 검토 메모

## 검토 결과

기존 화면은 기능은 충분하지만, 처음 사용하는 사람이 보기에는 버튼이 많고 실행 순서가 분산되어 있었다. 특히 Gmail/Google 관련 버튼이 상단에 한꺼번에 노출되어 “무엇을 먼저 눌러야 하는지”가 명확하지 않았다.

## 개선한 점

- 첫 화면에 `오늘 진행 순서`를 추가했다.
- 작업을 `명단 확인 -> 단계별 메일 -> 발송 승인 -> 미리보기 -> Gmail 결과` 순서로 배치했다.
- 각 단계 카드에 현재 상태와 바로 실행할 버튼을 함께 배치했다.
- 다음에 해야 할 작업을 `다음 작업` 문구와 강조 테두리로 표시한다.
- 기존 탭 구조는 유지하되, 세부 확인은 탭에서 하고 주요 실행은 첫 화면 순서 카드에서 하도록 정리했다.
- 비공개 Google Sheet 설정은 `Gmail 확인` 탭의 안내 박스에서 준비 상태를 확인하도록 했다.
- CLI에 있던 Gmail 발송 준비 파일 생성 단계를 웹 버튼으로 옮겨 운영 흐름이 끊기지 않도록 했다.
- Gmail 발송 준비 파일을 비공개 Google Sheet에 직접 올리는 버튼을 추가해 CSV 수동 업로드 단계를 줄였다.
- 엑셀/CSV 명단은 열 순서에 의존하지 않고 이메일 패턴과 제목 후보를 함께 보고 자동 판별하도록 바꿨다.
- 저장 전 `앱이 찾은 열`과 미리보기 5명을 보여주고, 사용자가 이메일/이름/메일 이름/퍼널 단계를 바로 바꿀 수 있게 했다.
- `메일 흐름`의 역할을 "메일 내용 + 다음 예약"으로 명확히 설명하고, 단계 추가/삭제와 `며칠 후`/`특정 날짜` 예약을 같은 화면에서 처리하도록 개선했다.
- Gmail 결과 반영 후 고객이 다음 메일 단계로 이동하도록 하여 사용자가 수동으로 상태를 바꾸지 않아도 다음 승인 목록을 만들 수 있게 했다.
- 명단 화면에서 체크박스로 보낼 사람을 고르고 `선택한 명단만 보내기`, `선택 삭제`, `전체 삭제`를 바로 실행할 수 있게 해 엑셀을 다시 만들지 않아도 운영 중 명단을 정리할 수 있게 했다.

## 남은 UX 개선 후보

- 실제 Google 계정 연결 후 오류 문구를 사용자 표현으로 더 다듬기
- 실제 사용자 엑셀 샘플로 자동 판별 정확도와 안내 문구 검증
- 확인 필요 항목만 모아 다시 처리하는 화면 추가
